# Risk Register

## Top 10 Critical Risks

1. Authentication exists in code, but production hardening and deployment controls remain incomplete.
2. Surrounding infrastructure hardening is explicitly out of scope for the base system.
3. SQLite remains part of the documented deployment model despite clear scale limits.
4. Integration testing is not implemented.
5. Performance testing is not implemented.
6. Backup strategy is listed as a production requirement, not a delivered capability.
7. System monitoring and alerting are incomplete.
8. Data cleanup and retention are only partially implemented.
9. The system is file-based and does not support realtime AIS feed ingestion.
10. The platform is explicitly not positioned as a complete maritime security system.
